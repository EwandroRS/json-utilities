Basic JSON Object Structure

A well-structured JSON object demonstrates fundamental data types and proper formatting conventions. This example shows a comprehensive user profile with various data types:

{
    "id": "USR_001",
    "firstName": "John",
    "lastName": "Doe",
    "email": "john.doe@example.com",
    "age": 30,
    "isActive": true,
    "salary": 75000.50,
    "department": null,
    "joinDate": "2023-01-15T08:30:00.000Z",
    "skills": ["JavaScript", "Python", "SQL"],
    "address": {
        "street": "123 Main St",
        "city": "New York",
        "zipCode": "10001",
        "country": "USA"
    },
    "preferences": {
        "theme": "dark",
        "language": "en-US",
        "notifications": {
            "email": true,
            "sms": false,
            "push": true
        }
    }
}

Key features: Mixed data types, nested objects, null values, proper date formatting (ISO 8601), and descriptive property names following camelCase convention.
